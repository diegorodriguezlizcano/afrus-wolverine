## 3. What to Build

### 3.1 Config Schema Validation

Create `src/config/env.validation.ts` using `class-validator` (already installed).

Define validation decorators for all environment variables:

```typescript
// Required (must be present — app fails to start without these):
- DATABASE_HOST: string
- DATABASE_PORT: number (valid port 1-65535)
- DATABASE_NAME: string
- DATABASE_USER: string
- DATABASE_PASSWORD: string
- JWT_SECRET: string (min length 20 chars)
- JWT_EXPIRES_IN: string (e.g. "7d", "24h")
- NODE_ENV: string (enum: development, staging, production)

// Required provider (must have at least one LLM key):
- LLM_PROVIDER: enum ('openrouter' | 'deepinfra')

// Required when LLM_PROVIDER=openrouter:
- OPENROUTER_API_KEY: string

// Required when LLM_PROVIDER=deepinfra:
- DEEPINFRA_API_KEY: string

// Optional (defaults provided):
- PORT: number (default: 3000)
- DATABASE_SSL: boolean (default: false for dev, true for prod)
- AFRUS_API_URL: string (default: 'https://backend.afrus.app')
- ALMA_WEBHOOK_URL: string (default: 'https://backend.afrus.app/api/v1/alma/webhook')
- LOG_LEVEL: string (default: 'info')
```

### 3.2 Config Service

Create `src/config/config.service.ts` using Nest.js `ConfigService` from `@nestjs/config`.

Provide typed getter methods:

```typescript
// Database
configService.getDatabaseUrl(): string  // builds postgres:// URL
configService.getDatabaseHost(): string
configService.getDatabasePort(): number
configService.getDatabaseName(): string
configService.getDatabaseUser(): string
configService.getDatabasePassword(): string

// LLM
configService.getLlmProvider(): 'openrouter' | 'deepinfra'
configService.getOpenRouterApiKey(): string | null
configService.getDeepInfraApiKey(): string | null
configService.getLlmApiKey(): string  // returns the active provider's key

// afrus
configService.getAfrusApiUrl(): string
configService.getAfrusApiKey(): string | null  // per-org keys come from DB

// ALMA
configService.getAlmaWebhookUrl(): string
configService.getAlmaWebhookSecret(): string | null

// JWT
configService.getJwtSecret(): string
configService.getJwtExpiresIn(): string

// App
configService.getPort(): number
configService.getNodeEnv(): string
configService.getLogLevel(): string
```

### 3.3 Config Module

Update `src/config/config.module.ts` (create if needed) that exports `ConfigModule` (re-exported) and `ConfigService`.

Mark the module `Global: true` so all other modules can inject `ConfigService` without re-importing.

### 3.4 Startup Validation

Integrate with `src/main.ts` to validate config on startup:

```typescript
// In main.ts — after NestFactory.create(), before app.listen()
const configService = app.get(ConfigService);
const errors = validateEnvironmentVariables();
if (errors.length > 0) {
  console.error('❌ Config validation failed:');
  errors.forEach(e => console.error(`  - ${e}`));
  process.exit(1);
}
```

Alternatively, use Nest.js `ValidationPipe` on the bootstrap or a custom `OnModuleInit` in `ConfigService`.
